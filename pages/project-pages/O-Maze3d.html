O-Maze 3D

O-Maze 3D is a simulation of orienteering where you have to navigate through a maze to find your way through controls. It is a mobile game developed for Android and is published on the Play Store.

Description

After my first Unity project "Gravity Race", I wanted to build a simulation of the sport I love doing most: Orienteering. Moving from 2D to 3D for the first time, I wanted to keep things rather simple and decided to use a labyrinth as an environment.

For this project, I had a clear vision of what features my game should include. Starting with the core features, I continued developing the app until it matched my vision.

Core Features

  • Game logic similar to orienteering
  • Map and course generator
  • GPS analysis
  • Night mode

Additional Features

  • Map and course editor mode
  • Sharing courses and files with friends
  • Online multiplayer mode
  • Automatic course analysis

Learnings

This project significantly expanded my experience compared to my previous one, as I worked with a larger codebase and more advanced features such as file handling, data storage, and an online multiplayer mode using Photon PUN.

Throughout the project, I developed a much better understanding of how to structure code in larger applications.
I learned the importance of starting with a simple working solution and gradually adding more complexity over time. Especially in larger scripts, I noticed a clear improvement in code quality, with less redundancy and more maintainable solutions.

Some of the main technical challenges included:

  • designing efficient ways to store and process the labyrinth
  • balancing accuracy and performance when tracking player movement
  • differentiating between tap and scroll for a smooth editor mode
  • implementing recursive algorithms for validating labyrinths and finding shortest routes
  • working with asynchronous processes and 3D-specific challenges such as rotations and spatial movement

Although I did not always fully understand the underlying theory at the time, I learned how to independently research, experiment, and apply solutions effectively.

What I Would Improve Today

Looking back, I would spend more time identifying reusable structures and refactoring existing code to build a cleaner and more maintainable codebase. Experiencing the drawbacks of duplicated code helped me understand the importance of reusability and clean structure in larger projects.
